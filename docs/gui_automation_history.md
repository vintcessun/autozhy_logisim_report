# Logisim GUI 自动化技术演进日志

本记录旨在追踪 Logisim 仿真报告系统的 GUI 交互层技术方案迭代。

---

## [2026-04-10] V1.0 - 物理坐标硬编码 (Initial)
- **实现方案**: 基于 `pyautogui.click(x, y)` 手动测量位置。
- **失败原因**: 极其脆弱。窗口位置变化、远程桌面缩放、不同版本的 UI 微调都会导致点击失效。

## [2026-04-10] V2.0 - 视觉驱动启动阶段 (Startup Vision)
- **实现方案**: 使用 UI-TARS (Ollama) 识别更新弹窗。
- **问题点**: 响应速度慢，模型在处理“有/否”这种二选一语义时定位尚可，但无法处理复杂的菜单项。

## [2026-04-10] V3.0 - 全屏语义操控尝试 (Full-Screen VLM)
- **实现方案**: 移除所有偏移量，全权交给 UI-TARS。
- **失败记录**:
    - **偏移故障**: 在 1920x1080 底图中，菜单文字（文件、编辑）像素占比极低，导致 AI 点击产生了 10-20 像素的垂直/水平偏移。
    - **系统菜单偏振**: 误点击左上角图标，触发系统窗口菜单而非业务菜单。

## [2026-04-10] V4.0 - 图像坐标映射映射修正 (API Cleanup)
- **实现方案**: 修复了 `ui-tars` 库返回小数坐标被 `pyautogui` 取整为 0 的底层 Bug。
- **成果**: 鼠标不再点击 `(0,0)`，但依然在“文件”和“编辑”之间徘徊，无法达到 100% 菜单级精度。

## [2026-04-10] V5.0 - OCR 增强与混合定位 (Proposed)
- **核心逻辑**: 
    - AI (UI-TARS) 理解指令意图（如：“我要点那个文件菜单”）。
    - OCR (RapidOCR) 负责在图像中精准扫描字符串所在的像素矩形框。
    - 混合引擎将 AI 的意图锁定到 OCR 探测到的物理中心。
- **预期目标**: 实现菜单操作的“指哪打哪”。
