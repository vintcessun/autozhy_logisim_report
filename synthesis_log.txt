# 16位并行进位加法器（CLA）设计规格说明书

**设计者**：Antigravity (Advanced Agentic Coding Team)
**目标**：实现16位组内并行、组间并行的快速加法器，解决串行进位（Ripple Carry）导致的延迟问题。

---

## 1. 核心逻辑公式调研与验证

### 1.1 基础信号生成（Bit Level）
对于每一位 $i$ ($0 \le i \le 15$):
- **Generate (生成信号)**: $g_i = A_i \cdot B_i$
- **Propagate (传递信号)**: $p_i = A_i \oplus B_i$
- **Sum (求和信号)**: $S_i = p_i \oplus C_i$

### 1.2 组内并行逻辑 (Intra-group Parallelism, 4-bit CLA)
每个4位块（Block）根据输入的 $C_{in}$ 立即计算出内部进位：
- $C_1 = g_0 + p_0 C_0$
- $C_2 = g_1 + p_1 g_0 + p_1 p_0 C_0$
- $C_3 = g_2 + p_2 g_1 + p_2 p_1 g_0 + p_2 p_1 p_0 C_0$
- **组产生信号 ($G^*$)**: $G_{0-3}^* = g_3 + p_3 g_2 + p_3 p_2 g_1 + p_3 p_2 p_1 g_0$
- **组传递信号 ($P^*$):** $P_{0-3}^* = p_3 \cdot p_2 \cdot p_1 \cdot p_0$

### 1.3 组间并行逻辑 (Inter-group Parallelism, LCU)
单设一个 **先行进位单元 (LCU)**，接收 4 个块的 $G^*$ 和 $P^*$，同时生成各组的进位：
- $C_4 = G_{0-3}^* + P_{0-3}^* C_0$
- $C_8 = G_{4-7}^* + P_{4-7}^* G_{0-3}^* + P_{4-7}^* P_{0-3}^* C_0$
- $C_{12} = G_{8-11}^* + P_{8-11}^* G_{4-7}^* + P_{8-11}^* P_{4-7}^* G_{0-3}^* + P_{8-11}^* P_{4-7}^* P_{0-3}^* C_0$
- $C_{16}$ (即最终 $C_{out}$) 按此类推。

---

## 2. 组件清单 (Component List)

| 模块名称 | 描述 | 关键元件 | 数量 |
| :--- | :--- | :--- | :--- |
| **G/P Generator** | 生成基础 $g_i, p_i$ | 与门 (AND), 异或门 (XOR) | 16组 |
| **4-bit CLA Block** | 4位子块，负责计算 $S_{0-3}, G^*, P^*$ | 组合逻辑门电路 | 4个 |
| **LCU (Level 2)** | 进位生成单元，计算 $C_4, C_8, C_{12}, C_{16}$ | 组合逻辑门电路 | 1个 |
| **Sum Logic** | 最终求和 | 异或门 (XOR) | 16个 |

---

## 3. 布线指导 (Wiring Guidance)

### 步骤 A：构建 4 位 CLA 子块
1. **输入引脚**：$A[3:0], B[3:0], C_{in}$。
2. **逻辑内部**：
   - 使用 XOR 得到 $p_{0-3}$，AND 得到 $g_{0-3}$。
   - 实现 1.2 节中的 $C_1, C_2, C_3$ 公式。
   - **注意**：$C_4$ 本身不由子块输出给下一级，子块只输出 $G^*$ 和 $P^*$ 给 LCU。
3. **输出引脚**：$S[3:0], G^*, P^*$。

### 步骤 B：连接 16 位顶层
1. **实例化**：放置 4 个“4位 CLA 子块”（命名为 Block 0, 1, 2, 3）。
2. **LCU 连接**：
   - 将 Block 0 的 $(G^*, P^*)$ 接到 LCU 的第 0 组输入。
   - 将 Block 1 的 $(G^*, P^*)$ 接到 LCU 的第 1 组输入，依此类推。
   - LCU 输入 $C_0$（总进位输入）。
3. **进位反馈（关键）**：
   - LCU 输出的 $C_4 \rightarrow$ Block 1 的 $C_{in}$。
   - LCU 输出的 $C_8 \rightarrow$ Block 2 的 $C_{in}$。
   - LCU 输出的 $C_{12} \rightarrow$ Block 3 的 $C_{in}$。
   - LCU 输出的 $C_{16} \rightarrow$ 整个加法器的 $C_{out}$。

---

## 4. 关键验证点

1. **时延分析**：在组间并行架构中，从 $C_0$ 到 $S_{15}$ 的门级时延仅为：
   - $G_i/P_i$ 生成 (1级) + 组 $G^*/P^*$ 生成 (2级) + LCU 生成 $C_i$ (2级) + 块内 $C_{i,k}$ 生成 (2级) + $S_i$ 异或 (1级) $\approx$ **8级门延迟**。
   - 相比于串行进位的 32+ 级延迟，性能大幅提升。
2. **常见错误**：
   - 不要将 Block 0 的 $C_{out}$ 串行连到 Block 1 的 $C_{in}$，这会退化为组间串行加法器。
   - $p_i$ 建议使用异或门实现（为了计算 Sum），若使用或门，则 $p_i$ 不能直接用于 $S_i$ 的异或计算。

---

class CLA16Bit:
    """
    16位并行进位加法器 (CLA) 
    架构：组内并行 (4-bit CLA) + 组间并行 (LCU)
    """

    class CLA4BitBlock:
        """4位进位产生/传递块"""
        def __init__(self, block_id):
            self.id = block_id

        def compute_gp(self, A_bits, B_bits):
            """第一层：生成 g[i] 和 p[i]"""
            g = [(A_bits[i] & B_bits[i]) for i in range(4)]
            p = [(A_bits[i] ^ B_bits[i]) for i in range(4)]
            return g, p

        def compute_group_signals(self, g, p):
            """计算组产生信号 G* 和组传递信号 P*"""
            # P* = p3 * p2 * p1 * p0
            P_star = p[3] & p[2] & p[1] & p[0]
            # G* = g3 + p3*g2 + p3*p2*g1 + p3*p2*p1*g0
            G_star = g[3] | \
                     (p[3] & g[2]) | \
                     (p[3] & p[2] & g[1]) | \
                     (p[3] & p[2] & p[1] & g[0])
            return G_star, P_star

        def compute_sum(self, A_bits, B_bits, Cin, g, p):
            """根据输入的进位 Cin 计算内部进位和最终和"""
            c = [0] * 4
            c[0] = Cin
            c[1] = g[0] | (p[0] & c[0])
            c[2] = g[1] | (p[1] & g[0]) | (p[1] & p[0] & c[0])
            c[3] = g[2] | (p[2] & g[1]) | (p[2] & p[1] & g[0]) | (p[2] & p[1] & p[0] & c[0])
            
            s = [p[i] ^ c[i] for i in range(4)]
            return s

    class LCU:
        """单设的先行进位单元 (Level 2)"""
        def compute_carries(self, G_stars, P_stars, C0):
            # C4 = G0* + P0*C0
            C4 = G_stars[0] | (P_stars[0] & C0)
            # C8 = G1* + P1*G0* + P1*P0*C0
            C8 = G_stars[1] | (P_stars[1] & G_stars[0]) | (P_stars[1] & P_stars[0] & C0)
            # C12 = G2* + P2*G1* + P2*P1*G0* + P2*P1*P0*C0
            C12 = G_stars[2] | (P_stars[2] & G_stars[1]) | \
                  (P_stars[2] & P_stars[1] & G_stars[0]) | \
                  (P_stars[2] & P_stars[1] & P_stars[0] & C0)
            # C16 = G3* + P3*G2* + P3*P2*G1* + P3*P2*P1*G0* + P3*P2*P1*P0*C0
            C16 = G_stars[3] | (P_stars[3] & G_stars[2]) | \
                  (P_stars[3] & P_stars[2] & G_stars[1]) | \
                  (P_stars[3] & P_stars[2] & P_stars[1] & G_stars[0]) | \
                  (P_stars[3] & P_stars[2] & P_stars[1] & P_stars[0] & C0)
            return [C4, C8, C12, C16]

    def __init__(self):
        self.blocks = [self.CLA4BitBlock(i) for i in range(4)]
        self.lcu = self.LCU()

    def add(self, A, B, Cin=0):
        """执行加法计算流程"""
        # 1. 将整数转换为位列表 (0-15 bit)
        A_bits = [(A >> i) & 1 for i in range(16)]
        B_bits = [(B >> i) & 1 for i in range(16)]

        # 2. 局部信号准备 (g, p, G*, P*)
        g_all, p_all = [], []
        G_stars, P_stars = [], []
        
        for i in range(4):
            b_a = A_bits[i*4 : i*4+4]
            b_b = B_bits[i*4 : i*4+4]
            g, p = self.blocks[i].compute_gp(b_a, b_b)
            G_s, P_s = self.blocks[i].compute_group_signals(g, p)
            g_all.append(g)
            p_all.append(p)
            G_stars.append(G_s)
            P_stars.append(P_s)

        # 3. LCU 并行生成进位 (这一步体现了组间并行)
        carries = self.lcu.compute_carries(G_stars, P_stars, Cin)
        # 进位分配：C0, C4, C8, C12
        block_cins = [Cin, carries[0], carries[1], carries[2]]
        Cout = carries[3]

        # 4. 各个 Block 根据 LCU 分配的进位并行计算 Sum
        final_s_bits = []
        for i in range(4):
            s_part = self.blocks[i].compute_sum(
                A_bits[i*4 : i*4+4], 
                B_bits[i*4 : i*4+4], 
                block_cins[i], 
                g_all[i], 
                p_all[i]
            )
            final_s_bits.extend(s_part)

        # 5. 合并结果
        sum_val = 0
        for i, bit in enumerate(final_s_bits):
            sum_val |= (bit << i)
            
        return sum_val, Cout

# 16位先行进位加法器 (CLA) 设计规格说明书

**版本：** 1.0 (Pro Edition)
**架构：** 组内并行 & 组间并行 (Hierarchical Carry Look-ahead)
**设计目标：** 消除 16 位加法中的逐位进位延迟，实现两级先行进位结构。

---

## 1. 逻辑公式定义 (Logic Formulas)

为了实现组内和组间的双重并行，我们将 16 位分为 4 个小组（Group 0-3），每组 4 位。

### 1.1 基础信号 (Bit-level)
对于每一位 $i$ ($i \in [0, 15]$):
- **生成信号 (Generate):** $g_i = A_i \cdot B_i$
- **传播信号 (Propagate):** $p_i = A_i \oplus B_i$
- **求和信号 (Sum):** $s_i = p_i \oplus C_i$

### 1.2 组级信号 (Group-level)
每个 4 位组（如 $i=0, 1, 2, 3$）产生两个关键信号：
- **组产生信号 ($G_G$):** $G_{G(0-3)} = g_3 + p_3g_2 + p_3p_2g_1 + p_3p_2p_1g_0$
- **组传播信号 ($P_G$):** $P_{G(0-3)} = p_3 \cdot p_2 \cdot p_1 \cdot p_0$

### 1.3 进位生成逻辑 (Carry Look-ahead Unit, LCU)
进位 $C_k$ 不再依赖前一位的输出，而是直接由 $G_G$ 和 $P_G$ 产生：
- $C_4 = G_{G0} + P_{G0}C_{in}$
- $C_8 = G_{G1} + P_{G1}G_{G0} + P_{G1}P_{G0}C_{in}$
- $C_{12} = G_{G2} + P_{G2}G_{G1} + P_{G2}P_{G1}G_{G0} + P_{G2}P_{G1}P_{G0}C_{in}$
- $C_{16} = G_{G3} + P_{G3}G_{G2} + \dots$ (最高位输出)

---

## 2. 模块化设计说明 (Components List)

在 Logisim (`.circ`) 中，建议封装为以下三个层次：

### 模块 A: 4位先行进位发生器 (CLA_Block_4bit)
- **输入：** $A[3:0]$, $B[3:0]$, $C_{in}$
- **输出：** $S[3:0]$, $P_G$, $G_G$
- **内部逻辑：** 包含 4 个全加器子逻辑，但不接力进位，而是生成 $P_G$ 和 $G_G$。

### 模块 B: 进位增加器 (Carry Look-ahead Unit, LCU)
- **输入：** $P_{G0,1,2,3}$, $G_{G0,1,2,3}$, $C_0$
- **输出：** $C_4, C_8, C_{12}$ (送回各 Block 作为输入), $C_{16}$ (最终进位)

### 模块 C: 16位顶层加法器 (Top_Adder_16bit)
- **组件清单：**
    - 4x `CLA_Block_4bit`
    - 1x `LCU`
    - 16-bit Input Pins (A, B)
    - 16-bit Output Pins (Sum)

## 4. 关键自检点 (Troubleshooting)

1.  **进位环路：** 确保 $C_4$ 是由 LCU 算出来的，而不是从 `Block 0` 的内部串行传递出来的。如果 `Block 0` 内部有 $C_4$ 的 Ripple 输出，请将其悬空。
2.  **异或门延迟：** 全加器的 $S$ 位必须由 $P_i \oplus C_i$ 生成，确保 $C_i$ 是来自该组对应的 Carry Buffer。
3.  **16位分段：** 
    - Block 0: 处理 bits 0-3
    - Block 1: 处理 bits 4-7
    - Block 2: 处理 bits 8-11
    - Block 3: 处理 bits 12-15

# 16位快速加法器 (CLA) 设计规格说明书 (Pro版)

**设计目标**：实现一个 16 位先行进位加法器（Carry Lookahead Adder），通过“组内并行”和“组间并行”两级架构，将进位传递延迟降低至 $O(\log n)$ 级别。

---

### 一、 核心逻辑公式调研与验证

基于数字电子技术标准，16位 CLA 采用 **4位一组 (Block)** 的分层结构。

#### 1. 位级信号 (Bit-level)
对于每一位 $i$ ($i=0 \dots 15$):
- **生成信号 (Generate)**: $g_i = A_i \cdot B_i$
- **传递信号 (Propagate)**: $p_i = A_i \oplus B_i$
- **本位和 (Sum)**: $S_i = p_i \oplus C_i$

#### 2. 4位组内并行逻辑 (Intra-group, 4-bit CLA)
以第 0 组为例（包含 bits 0-3），进位 $C_1, C_2, C_3$ 直接由 $C_0$ 计算：
- $C_1 = g_0 + p_0 C_0$
- $C_2 = g_1 + p_1 g_0 + p_1 p_0 C_0$
- $C_3 = g_2 + p_2 g_1 + p_2 p_1 g_0 + p_2 p_1 p_0 C_0$
- **组生成信号 (Group Generate)**: $G_0 = g_3 + p_3 g_2 + p_3 p_2 g_1 + p_3 p_2 p_1 g_0$
- **组传递信号 (Group Propagate)**: $P_0 = p_3 \cdot p_2 \cdot p_1 \cdot p_0$

#### 3. 组间并行逻辑 (Inter-group, LCU)
Lookahead Carry Unit (LCU) 接收各组的 $G, P$ 信号，计算跨组进位：
- $C_4 = G_0 + P_0 C_0$
- $C_8 = G_1 + P_1 G_0 + P_1 P_0 C_0$
- $C_{12} = G_2 + P_2 G_1 + P_2 P_1 G_0 + P_2 P_1 P_0 C_0$
- $C_{16} = G_3 + P_3 G_2 + P_3 P_2 G_1 + P_3 P_2 P_1 G_0 + P_3 P_2 P_1 P_0 C_0$ (即末位进位 $C_{out}$)

| **PG_Gen** | $A[3:0], B[3:0]$ | $g[3:0], p[3:0]$ | 计算4位原始 P、G 信号 |
| **CLA_4bit** | $g[3:0], p[3:0], C_{in}$ | $S[3:0], G, P$ | 组内逻辑，不输出中间进位到顶层，仅输出组 PG |
| **LCU_16bit** | $G_{0-3}, P_{0-3}, C_0$ | $C_4, C_8, C_{12}, C_{16}$ | 全局进位逻辑（核心大脑） |

### 三、 布线指导 (Wiring Guidance)

#### 层级拓扑：
1. **第一层 (底层)**：将 16 位输入分成 4 个 **4-bit CLA 组**。
2. **第二层 (中层)**：放置 1 个 **LCU 模块**。
3. **数据流向**：
   - 各组生成的 $G_j, P_j$ ($j=0 \dots 3$) 送入 LCU。
   - LCU 计算出 $C_4, C_8, C_{12}$ 后，“回传”给后续组作为其 $C_{in}$。
   - $C_0$ 直接送入 LCU 和第 0 组。
   - $C_4$ 送入第 1 组， $C_8$ 送入第 2 组， $C_{12}$ 送入第 3 组。

#### 常见错误规避：
- **禁止串行**：严禁将第 0 组的 $C_4$ 串联到第 1 组。必须由 LCU 并行计算后同时分发。
- **异或实现**：$p_i$ 建议使用异或门，这样在计算 $S_i$ 时可以复用。
- **LCU 延迟**：LCU 模块内部从 $C_0$ 到 $C_{16}$ 的门级延迟应恒定。

---

### 四、 性能分析 (Performance)

*   **全加器 (Ripple Carry)**: 延迟约为 $16 \times 2 = 32$ 级门延迟。
*   **本 16位 CLA**:
    1. 生成 $g_i, p_i$: 1 级。
    2. LCU 生成组进位 $G/P$: 2 级。
    3. LCU 生成 $C_4 \dots C_{16}$: 2 级。
    4. 组内计算 $C_{1,2,3}$ 并求和: 2 级。
    *   **总计**: 约 **7-8 级门延迟**，速度提升约 4 倍。

**版本：** 1.0 (Pro Edition)
**架构类型：** 组内并行 (Intra-group Parallel) + 组间并行 (Inter-group Parallel)

## 1. 逻辑公式调研与验证

### 1.1 一级：位进位逻辑 (Bit-level Logic)
对于每一位 $i \in [0, 15]$：
- **生成信号 (Generate):** $g_i = A_i \cdot B_i$
- **传播信号 (Propagate):** $p_i = A_i \oplus B_i$
- **和 (Sum):** $S_i = p_i \oplus C_i$

### 1.2 二级：4位组内先行进位 (CLA4 Block)
将16位分为4组（Group 0-3），每组内部并行计算进位：
- $C_1 = g_0 + p_0C_0$
- $C_2 = g_1 + p_1g_0 + p_1p_0C_0$
- $C_3 = g_2 + p_2g_1 + p_2p_1g_0 + p_2p_1p_0C_0$
- **组生成信号 ($G_G$):** $G_G = g_3 + p_3g_2 + p_3p_2g_1 + p_3p_2p_1g_0$
- **组传播信号 ($P_G$):** $P_G = p_3p_2p_1p_0$

### 1.3 三级：组间先行进位单位 (LCU - Lookahead Carry Unit)
LCU 接收各组的 $G_{Gj}$ 和 $P_{Gj}$，实现组间的并行进位：
- $C_4 = G_{G0} + P_{G0}C_{in}$
- $C_8 = G_{G1} + P_{G1}G_{G0} + P_{G1}P_{G0}C_{in}$
- $C_{12} = G_{G2} + P_{G2}G_{G1} + P_{G2}P_{G1}G_{G0} + P_{G2}P_{G1}P_{G0}C_{in}$
- $C_{16} = G_{G3} + P_{G3}G_{G2} + P_{G3}P_{G2}G_{G1} + P_{G3}P_{G2}P_{G1}G_{G0} + P_{G3}P_{G2}P_{G1}P_{G0}C_{in}$

## 3. 布线指导 (Wiring Guidance)

### 3.1 分层拓扑
1. **底层 (Leaf):** A[15:0] 和 B[15:0] 接入 PG 生成器。
2. **中层 (Group):**
   - 每 4 位 PG 信号组进入一个 **CLA4 子电路**。
   - ⚠️ **注意：** CLA4 子电路内部**不应**直接计算 $C_4, C_8$ 等跨组信号，而是输出 $P_G, G_G$。
3. **顶层 (LCU Interconnect):**
   - CLA4_0 ($P_{G0}, G_{G0}$) -> LCU 端口 0
   - CLA4_1 ($P_{G1}, G_{G1}$) -> LCU 端口 1
   - ...以此类推。
   - LCU 的输出 $C_4, C_8, C_{12}$ 分别反馈给 CLA4_1, CLA4_2, CLA4_3 的入位进位端口。

### 3.2 关键路径优化
- 确保所有的 $g_i, p_i$ 并行生成。
- 进位链的总延迟 = $T_{pg} + T_{LCU\_pg} + T_{LCU\_carry} + T_{xor\_sum}$。
- 在 Logisim 等仿真器中，避免使用“隧道 (Tunnel)”过多导致逻辑链路不清晰，建议使用总线 (Bus/Splitter) 管理 16 位输入。
