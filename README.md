## autozhy_logisim_report

**autozhy_logisim_report** 是一个基于多智能体（Multi-Agent）协作架构的自动化实验助手，专为计算机组成原理课程设计。它利用先进的视觉语言模型（VLM）与大语言模型（LLM）技术，实现对 **Logisim** 仿真软件的深度自动化操控、电路逻辑验证与设计，并最终一键生成符合学术规范的 Markdown 实验报告。

------

### 🚀 核心愿景

在传统的计算机组成原理实验中，学生往往需要耗费大量时间在重复的拨码验证、手动截图和繁琐的文档排版上。本项目旨在通过 AI 智能体模拟人类的“手”与“眼”，将复杂的实验流程自动化，让学生能够更专注于底层架构的设计逻辑。

------

### 🧠 智能体矩阵 (Agent Matrix)

本项目由四个核心智能体协同工作，形成一个闭环的自动化流水线：

- **内容解析智能体 (Content Parsing Agent)**：负责对嵌套压缩包进行递归解压，并从 PDF 指导书与 DOCX 样本中精准提取实验要求、环境说明与固定模板。
- **设计性实验智能体 (Design Experiment Agent)**：采用 **Actor-Critic** 闭环架构。子智能体负责生成 `.circ` XML 代码并进行拓扑可达性校验，主智能体负责逻辑审查，实现“设计-修正-验证”的纯代码级闭环。
- **验证性实验智能体 (Verification Experiment Agent)**：原生集成 **ui-tars** 视觉模型与 **PyAutoGUI** 操控引擎，负责在物理机上模拟真实点击、触发时钟脉冲，并截取实验结果数据。
- **实验报告智能体 (Report Writing Agent)**：作为全局调度中枢（Orchestrator），负责统筹实验进度，并调用轻量级模型对结果进行精简分析，最后按照规范模板导出带有相对路径配图的 `.md` 报告。

------

### ✨ 技术特性

- **异构模型协作**：由 `gemini-3.1-pro` 负责核心推理，`gemini-3-flash` 负责快速排版，本地 `ui-tars` 负责屏幕解析，实现效能最优解。
- **视觉驱动操控**：无需修改 Logisim 源码，直接通过视觉模型识别 UI 组件坐标，模拟人类真实操作。
- **代码级设计闭环**：在电路生成阶段通过静态拓扑扫描（Linting）而非视觉反馈进行修正，极大提升了设计实验的成功率。
- **OS 级互斥调度**：内置物理屏幕资源 Mutex 锁与 DPI 感知修正，确保多任务并行时窗口置顶与点击定位的绝对准确。
- **标准化输出**：生成的报告严格对齐实验样本格式，图片自动存入 `.assets` 目录，支持 Typora 等编辑器完美渲染。

### 🛠️ 环境配置 (Environment Setup)

本项目依赖以下环境与外部工具，请在运行前确保其已正确安装并加入系统环境变量：

#### 1. 软件依赖
- **Python 3.12+**: 核心运行环境。
- **uv**: 推荐的 Python 包及虚拟环境管理器。
- **Logisim-evolution**: 实验仿真软件，建议保持窗口缩放为默认值。

#### 2. 外部二进制工具 (通过 patool 调用)
由于 `Content Parsing Agent` 需要处理多种格式的压缩包（如 `.rar`, `.7z`），请确保安装以下工具：
- **7-Zip** (推荐): 安装后确保 `7z.exe` 路径已加入系统的 `Path` 环境变量。
- 或其他 `patool` 支持的解压工具（如 WinRAR）。

#### 3. 硬件要求
- **NVIDIA GPU**: 为了本地运行 `ui-tars` 视觉模型，建议显存 $\ge$ 8GB。

#### 4. 快速开始
```bash
# 同步依赖并创建虚拟环境
uv sync

# 运行主程序（待开发）
# python main.py
```
