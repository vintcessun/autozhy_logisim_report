# 生成物技术规范 (SPEC.md)

本文件定义了 `autozhy_logisim_report` 自动化生成的实验报告及其关联附件的质量基准与规范。

------

## 1. 最终基准 (Baseline)

所有生成的实验报告必须以 `docs/examples/example_output.md` 为视觉与结构基准。

### 1.1 结构基准

- **无损迁移区**：#1 实验环境与 #2 实验目的 必须与原始 `.docx` 样本保持字符级一致。
- **分段逻辑**：
  - **验证性实验 (3.1)**：采用 `### (n) 题目名` -> `#### 实验结果` (配图) -> `#### 实验分析` (文字) 的三级嵌套。
  - **设计性实验 (3.2/3.3)**：在“实验结果”前强制插入 `#### 电路设计` 模块。
- **引用规范**：图片必须使用 Markdown 原生语法引用，路径统一为相对路径 `./{实验名}.assets/{文件名}.png`。

### 1.2 视觉基准

- **实验结果图**：必须是局部裁剪图（仅保留探针、数码管、LED等数据输出区域），禁止包含无关的工具栏或逻辑门背景。
- **电路设计图**：必须是全局全景图。Agent 在截图前需通过 `ui-tars` 指令执行 `Ctrl+0` (适应屏幕) 或调整缩放比例，确保逻辑拓扑清晰可见。

------

## 2. 过程产物 (Artifacts)

在自动化执行过程中，`debug/` 文件夹下会产生以下中间件，用于系统溯源：

| **中间件名称**                    | **存放路径**            | **说明**                                                     |
| --------------------------------- | ----------------------- | ------------------------------------------------------------ |
| **任务蓝图 (Task JSON)**          | `debug/task_list.json`  | 由 `Content Parsing Agent` 生成，包含从 PDF 提取的所有实验动作序列。 |
| **拓扑快照 (Temp Circ)**          | `debug/xml_diff/`       | 记录 `Design Agent` 在 Actor-Critic 闭环中每一轮迭代生成的原始 XML 片段。 |
| **视觉坐标记录 (Grounding JSON)** | `debug/screenshot/`     | `ui-tars` 返回的原始点击坐标与对应的带框识别截图。           |
| **原始提取文本 (Raw Text)**       | `debug/raw_extract.txt` | 从 PDF/DOCX 中初次剥离的原始文本，用于排版 Agent 进行 Context 对比。 |

## 3. 故障排除与质量修复 (Troubleshooting)

若生成的报告出现异常，应通过以下逻辑进行调整：

### 3.1 图像比例与清晰度异常

- **现象**：截图模糊、文字重叠或图片边缘切口。
- **修复逻辑**：
  1. 检查 `DEVELOPMENT.md` 中的 **DPI Awareness** 是否生效（物理像素 1:1 映射）。
  2. 调整 `Verification Agent` 的截图 Padding 参数。
  3. 强制 `pygetwindow` 重新执行 `maximize()` 动作，确保渲染窗口尺寸固定。

### 3.2 实验分析逻辑幻觉

- **现象**：分析文本与图片数据不符（如图片显示缺失率 25%，文字写成 10%）。
- **修复逻辑**：
  1. 调高 `Report Agent` 的温度系数（Temperature）至更稳定的低值。
  2. 在 Prompt 中强制要求模型执行 **视觉验证步骤**：*“先描述图中看到的数字，再基于该数字进行逻辑推导”*。

### 3.3 Markdown 排版错乱

- **现象**：图片链接失效或标题层级错误。
- **修复逻辑**：
  1. 校验 `output/` 文件夹名是否包含非法字符（如空格、百分号），建议使用 `snake_case`。
  2. 检查子智能体返回的 `analysis_raw` 文本是否自带了冲突的 `#` 标题，若有，由 `gemini-3-flash` 进行正则清洗。

## 4. 自动化 Double Check 检查单

`Report Agent` 在最终落盘前，需针对每个实验部分对照此单：

- [ ] 图片路径是否存在于 `.assets/` 文件夹中？
- [ ] 验证性实验是否已经剔除掉冗余的电路背景图？
- [ ] 设计性实验是否包含完整的电路拓扑图？
- [ ] 思考题是否已按照 `.docx` 的顺序精准插入？
- [ ] 所有的布尔控制类变量类名是否符合 `EnableCard` 规范
